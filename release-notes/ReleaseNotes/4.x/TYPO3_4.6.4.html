Release Notes for TYPO3 4.6.4

This document contains information about TYPO3 version 4.6.4 which was released on January 24th 2012.

News

This version is a maintenance release and contains bug fixes only.

Upgrading

The usual upgrading procedure applies. No database updates are necessary.

Changes

Here is a list of what was fixed since 4.6.3:

  • [RELEASE] Release of TYPO3 4.6.4
  • [BUGFIX] typolink prepends wrong domain when using url scheme (#31771)
  • [BUGFIX] Regression in language handling (#33227)
  • [BUGFIX] htmlArea RTE: magic image maxWidth is not working as expected (#32301)
  • [BUGFIX] htmlArea RTE : Adding link problem with IE (#31763)
  • [BUGFIX] PHP warning if open_basedir is enabled (#32109)
  • [BUGFIX] TCEmain::clear_cacheCmd relies on active BE_USER (#28007)
  • [BUGFIX] MENU special = updated does not work anymore (#32374)
  • [BUGFIX] Usage of custom caching backends (#32986)
  • [BUGFIX] HTML entity   is inserted before and after inline element (#32263)
  • [BUGFIX] Module menu link wrap (#32448)
  • [BUGFIX] ENABLE_INSTALL_TOOL does not respect fileCreateMask (#21740)
  • [BUGFIX] Fix permissions of downloaded translations into l10n folder (#31964)
  • [BUGFIX] Vague error message in t3lib_div (#18545)
  • [BUGFIX] Link to mounted shortcut page lacks &MP parameter (#32938)
  • [BUGFIX] Typo in Install Tool (#32970)
  • [TASK] Scheduler: Add “Add Task” button to header (#32753)
  • [BUGFIX] Reports are called twice (#32768)
  • [BUGFIX] locallangXMLOverride is broken since the switch to XLIFF (#32573)
